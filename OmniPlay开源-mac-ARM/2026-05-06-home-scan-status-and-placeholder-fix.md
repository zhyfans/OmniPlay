# 2026-05-06 首页扫描状态与占位海报修正

## 修正内容

- 首页左上角状态优先显示扫描/刮削进度；添加媒体源后到海报出现前，会显示“准备扫描”“扫描目录中”“扫描中”等状态，不再空白。
- 扫描入库时只为能提取出可用影视名称的文件创建首页占位；纯发布参数、通用目录名、无法识别标题的文件会跳过，不再出现在首页。
- 剧集文件使用提取出的剧名作为占位聚合键，同一部剧的多集文件会归到一个首页海报。
- 移除媒体源时会取消该源的分集剧照任务，并取消当前扫描/刮削流程，左上角改为“正在移除媒体源...”，避免继续显示旧的刮削或获取剧照状态。
- 旧库中已经存在的负 ID 占位会在首页读取时再次过滤，避免历史无效占位继续显示。

## 验证

- 已通过 `xcodebuild -project OmniPlay.xcodeproj -scheme OmniPlay -configuration Debug -sdk macosx build`。
