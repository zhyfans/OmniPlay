# 2026-05-06 扫描后即时刮削顺序修正

## 修正内容

- 扫描入库流程改为按影视组处理：提取出一部电影，或提取出包含全部分集的同一部剧集后，先写入该组占位条目，再立即刮削该电影/剧集的元数据和海报。
- 不再等同一媒体源内所有文件都完成入库后才开始刮削；首页可随着每个影视组完成刮削逐步显示真实海报。
- 未刮削条目按占位 `movieId` 分组处理；同一部剧集的所有分集只用代表文件查询一次 TMDB，成功后整组分集一起关联到同一个剧集条目。
- 剧集元数据/海报成功后不再立刻启动分集剧照下载，避免分集剧照阻塞后续电影和剧集海报显示。
- 分集剧照保留在所有元数据和海报刮削完成后统一补抓。

## 验证

- 已通过 `xcodebuild -project OmniPlay.xcodeproj -scheme OmniPlay -configuration Debug -sdk macosx -destination platform=macOS,arch=x86_64 build`。
