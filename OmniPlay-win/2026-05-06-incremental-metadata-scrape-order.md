# 2026-05-06 扫描后即时刮削顺序修正

## 修正内容

- Windows 扫描器新增影视组入库回调：提取出一部电影，或提取出包含全部分集的同一部剧集后，先提交该组入库，再立即回调首页流程刮削该电影/剧集的元数据和海报。
- 首页同步不再等同一媒体源内全部文件入库后才开始刮削；新电影和新剧集会随着扫描入库逐个刷新海报。
- 新增本地、WebDAV、Plex、Emby、Jellyfin、SMB 挂载和重新开启媒体源时，会优先处理当前新增/开启的源，不再固定等待所有媒体源扫描完成。
- 自动刮削顺序拆分为两段：先完成全部电影和剧集的元数据/海报，再统一处理分集剧照。
- 分集剧照不再插入每部剧集的元数据刮削循环中，避免阻塞后续海报显示。

## 验证

- 已通过 `dotnet build OmniPlay-win/OmniPlay.Windows.slnx -p:EnableWindowsTargeting=true -p:NuGetAudit=false`。
- 已通过 Windows 扫描/标题提取相关定向测试 16 项。
